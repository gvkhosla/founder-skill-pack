# Canonical Sequences Source

Sequences are multi-step workflows that tie skills together.

Examples:
- `validate-to-build`
- `build-to-launch`
- `gtm-engine`
- `pmf-recovery`
- `weekly-operating-rhythm`

Each sequence should contain:
- `sequence.yaml`
- `steps.md`
- `outputs.schema.json`
- optional host-specific prompt helpers
